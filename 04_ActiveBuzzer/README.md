# Project: 04_ActiveBuzzer

## 1. Description
This project focuses on the use of the `tone()` and `noTone()` functions to control an Active Buzzer. We explore the electrical differences between a simple component and a 3-pin powered module, emphasizing correct voltage distribution and frequency management.

## 2. Learning Objectives
* Properly wire a 3-pin electronic module (VCC/GND/Signal).
* Master the `tone()` function to generate square waves.
* Use `noTone()` to manage silence intervals precisely.

## 3. Hardware Requirements
* **Microcontroller:** Arduino Uno R3.
* **Actuator:** 1x Active Buzzer (Component or 3-pin Module).
* **Jumpers:** Male-to-Male or Male-to-Female.

## 4. Connection Details
The 3-pin module requires a constant power source to feed its internal transistor.



| Module Pin | Arduino Pin | Function |
| :--- | :--- | :--- |
| **VCC (+)** | 5V | Constant Power (feeds the oscillator) |
| **GND (-)** | GND | Ground Reference |
| **S / Signal** | Pin 4 | Control Signal (carries the frequency) |

## 5. Expected Behavior
The buzzer will produce a 1000Hz tone for 500ms and remain silent for 1000ms. If using a module, the sound is generated by the 5V rail but controlled by the signal from Pin 4.

## 6. Technical Explanation: tone() and noTone()

### **The tone() Function**
The `tone(pin, frequency)` function is used to generate a square wave. 
* **Frequency:** Measured in Hertz (Hz). It determines how many times per second the pin toggles between 5V and 0V.
* **Internal Mechanism:** It uses the Arduino's internal **Hardware Timers**. This is why it can run in the background while the code continues to the next line (unless a duration is specified).
* **Pin Flexibility:** It can be used on **any digital pin**, as the timer can be mapped to any output.

### **The noTone() Function**
The `noTone(pin)` function is the only way to stop a tone started without a fixed duration.
* **Precision:** While `digitalWrite(LOW)` simply cuts the voltage, `noTone()` deactivates the internal timer associated with that pin. 
* **Importance:** Using `noTone()` prevents "ghost" sounds or interference when the buzzer should be completely silent.
