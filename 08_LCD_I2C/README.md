# Project 08: Human-Machine Interface (HMI) via LCD I2C

## 1. Description
This project focuses on the implementation of **Data Visualization** in robotics systems. In advanced engineering, moving beyond simple LED status indicators is crucial. We use a **16x2 Liquid Crystal Display (LCD)** integrated with an **I2C interface** to provide detailed alphanumeric feedback, system telemetry, and custom branding, all while optimizing hardware resources.

## 2. Hardware Anatomy: How an LCD Works
The LCD is a passive optical device based on the twisted nematic (TN) liquid crystal principle.
* **Backlight:** Provides the light source (LED panel).
* **Liquid Crystals:** These do not emit light; instead, they change their orientation when an electric field is applied, acting as shutters that block or allow light to pass through polarizing filters to create the dark pixels visible on the screen.
* **Character Grid:** A 16x2 display provides 32 slots. Each slot is a matrix of 5x8 individual pixels.

## 3. Communication Protocol: I2C (Inter-Integrated Circuit)
Standard LCD wiring requires at least 6 to 10 digital pins. The **I2C Backpack (PCF8574 chip)** allows the Arduino to control the entire screen using only two wires:
* **SDA (Serial Data):** Bidirectional line for data transmission.
* **SCL (Serial Clock):** Synchronization signal generated by the Master (Arduino).
* **Hex Addressing:** The LCD responds to the specific address `0x27`, allowing the I2C bus to support multiple sensors on the same pins.

## 4. Library and Software Methods (`LiquidCrystal_I2C.h`)
The firmware uses high-level abstraction methods to manage the display:
* **`lcd.init()`**: Configures the I2C communication and resets the internal controller.
* **`lcd.backlight()`**: Switches the transistor that powers the LED panel.
* **`lcd.setCursor(col, row)`**: Moves the memory pointer to specific coordinates (0-indexed).
* **`lcd.createChar(slot, array)`**: Writes a bit-map directly into the **CGRAM** (Character Generator RAM).
* **`lcd.write(slot)`**: Sends raw data from one of the 8 available CGRAM memory slots to the screen.
* **`lcd.scrollDisplayLeft() / Right()`**: Shifts the display window across the internal buffer.

## 5. Custom Graphics and Icon Design
Students can design up to 8 unique icons (batteries, sensors, logos) by mapping a 5x8 pixel grid in binary notation (1 for ON, 0 for OFF).
* **Online Design Tool:** [LCD Custom Character Generator](https://maxpromer.github.io/LCD-Character-Creator/)
* Use this tool to generate the `byte` array required for your code.

## 6. Power Infrastructure & Safety Protocol
According to our hardware guidelines:
1. **Breadboard Powering:** All peripherals must be powered from the breadboard's 5V/GND rails to ensure current stability.
2. **Common Ground:** The system must share a common GND reference to prevent erratic data behavior.
3. **Contrast Calibration:** The physical blue potentiometer (trimmer) on the back of the I2C module must be adjusted manually until the characters are sharp.

## 7. Hardware Requirements
* **Microcontroller:** Arduino Uno R3.
* **Output:** 16x2 LCD with I2C Module.
* **Cabling:** 4 Female-to-Male Jumpers.
